# Testing Fault Tolerance

Here we show a few examples of explicitly testing the fault tolerance of the [[5,1,3]] codepack.

## Non-FT Program

Our starting point will be the non-FT program from the [Workflow tutorial](../gettingstarted/workflow).

```{code-cell} ipython3
from loqs.backends import QSimQuantumState, PyGSTiNoiseModel, PyGSTiPhysicalCircuit
from loqs.core import QuantumProgram
from loqs.codepacks import codepack_5_1_3_quantinuum2022 as codepack_5_1_3

code_5q = codepack_5_1_3.create_qec_code(circuit_backend=PyGSTiPhysicalCircuit)

# Define the physical qubit labels for our system here
qubits = ["A0", "A1"] + [f"D{i+2}" for i in range(5)]

ideal_model = codepack_5_1_3.create_ideal_model(qubits, model_backend=PyGSTiNoiseModel)

state_type = QSimQuantumState
patch_types = {"5Q": code_5q}
```

```{code-cell} ipython3
# Stack
# The "Init *" instructions will be generated by the QuantumProgram based on the values
# of state_type and patch_types below
stack = [
    ("Init State", None, (len(qubits),), {"qubit_labels": qubits}), # Autogenerated from state_type
    ("Init Patch 5Q", None, ("L0", qubits)), # Autogenerated from patch_types. Note that 5Q here must be a key into the patch_types dict below
    ("Non-FT Minus Prep", "L0"),
    ("Non-FT Logical X Measure", "L0")
]

program = QuantumProgram(
    stack,
    default_noise_model=ideal_model,
    state_type=state_type,
    patch_types=patch_types,
    name="Prep minus, measure X"
)
```

```{code-cell} ipython3
# And now we can run it for real!
program.run(shots=100)

from collections import Counter
Counter(program.collect_shot_data("logical_measurement", -1))
```

## Sanity Check: Logical Minus Prep

While debugging, I found being able to print out the state in terms of the computational basis useful.
I've added this functionality to `loqs.internal.qsimtools`.

We can use this functionality to verify that the logical minus prep results in the correct state (given in comments, worked out by hand from the encoding circuit).

```{code-cell} ipython3
from loqs.internal.qsimtools import print_state_probs_phases
```

```{code-cell} ipython3
stack_prep_check = [
    ("Init State", None, (len(qubits),), {"qubit_labels": qubits}), # Autogenerated from state_type
    ("Init Patch 5Q", None, ("L0", qubits)), # Autogenerated from patch_types. Note that 5Q here must be a key into the patch_types dict below
    ("Non-FT Minus Prep", "L0"),
]

program_prep_check = QuantumProgram.from_quantum_program(program, stack_prep_check, name="Prep -")

program_prep_check.run()

hist = program_prep_check.shot_histories[-1]
final_state = hist[-1]["state"]
print_state_probs_phases(final_state)

# An equal superposition of all 0000000 -> 0011111 in lexographical order with phases:
# Starting at 0000000: +, +, +, -, +, +, -, +
# Starting at 0001000: +, +, +, -, -, -, +, -
# Starting at 0010000: +, -, +, +, +, -, -, -
# Starting at 0011000: -, +, -, -, +, -, -, -
# This appears correct, i.e. we are initializing logical - correctly
```

## Sanity Check: Stabilizers

Stabilizers should leave the state unchanged. So we should be able to do a non-FT prep, each stabilizer, and then a non-FT unprep and get all 0s out.

```{code-cell} ipython3
for stab in ["XZZXI", "IXZZX", "XIXZZ", "ZXIXZ"]:
    stack_stab = [
        ("Init State", None, (len(qubits),), {"qubit_labels": qubits}), # Autogenerated from state_type
        ("Init Patch 5Q", None, ("L0", qubits)), # Autogenerated from patch_types. Note that 5Q here must be a key into the patch_types dict below
        ("Non-FT Minus Prep", "L0"),
        (f"{stab} Stabilizer", "L0"),
        ("Non-FT Minus Unprep", "L0")
    ]

    program_stab = QuantumProgram.from_quantum_program(program, stack_stab, name=f"Prep -, Stab {stab}, Unprep -")

    program_stab.run()

    hist = program_stab.shot_histories[-1]
    final_state = hist[-1]["state"]

    print(f"{stab} Probabilities")
    print_state_probs_phases(final_state)
    print()
```

## FT Prep with Single Errors

We will test FT prep by inserting an X or Z error on each qubit prior to the FT prep.
We'll do this by creating a physical circuit `Instruction` with the X or Z gate on the appropriate qubit, and inserting that into the stack before FT prep.
Note: I actually want to have a much more sophisticated FT testing framework, but this is the dirty hacked version.

What should happen is that one round of prep will occur and fail, but then second round should succeed.

```{code-cell} ipython3
from loqs.core.instructions import builders
```

```{code-cell} ipython3
for gate in ["Gxpi", "Gzpi"]:
    for qubit in qubits:
        # Create a global error
        error = builders.build_physical_circuit_instruction(
            PyGSTiPhysicalCircuit([(gate, qubit)], qubit_labels=qubits),
            name=f"{gate[1].upper()} error on {qubit}"
        )

        stack_ftprep_error = [
            ("Init State", None, (len(qubits),), {"qubit_labels": qubits}),
            ("Init Patch 5Q", None, ("L0", qubits)),
            ("Error", None),
            ("FT Minus Prep", "L0"),
            ("Non-FT Logical X Measure", "L0")
        ]

        program_ftprep_error = QuantumProgram.from_quantum_program(
            program,
            stack_ftprep_error,
            global_instructions={'Error': error},
            name=f"{gate[1].upper()} error on qubit {qubit}, FT Prep -, measure X")

        program_ftprep_error.run(shots=100)

        from collections import Counter
        print(Counter(program_ftprep_error.collect_shot_data("logical_measurement", -1)))

# Not sure what is happening with X errors on D4 and D6.
# Somehow that is turning into a logical error...
# Still, everything else is working (including meas/flag errors, so worth playing with!)
```

## QEC with Single Errors

This time, we'll insert errors AFTER the prep and let QEC catch them.

Note that this is UNFLAGGED, so we expect to not always correct errors on measurement/flag qubits (in fact, these will sometimes turn into logical errors).
Just so happens that we luck out for this test case.

```{code-cell} ipython3
for gate in ["Gxpi", "Gzpi"]:
    for qubit in qubits:
        # Create a global error
        error = builders.build_physical_circuit_instruction(
            PyGSTiPhysicalCircuit([(gate, qubit)], qubit_labels=qubits),
            name=f"{gate[1].upper()} error on {qubit}"
        )

        stack_qec_error = [
            ("Init State", None, (len(qubits),), {"qubit_labels": qubits}), # Autogenerated from state_type
            ("Init Patch 5Q", None, ("L0", qubits)), # Autogenerated from patch_types. Note that 5Q here must be a key into the patch_types dict below
            ("Non-FT Minus Prep", "L0"),
            ("Error", None),
            ("Unflagged QEC", "L0"),
            ("Non-FT Logical X Measure", "L0")
        ]

        program_qec_error = QuantumProgram.from_quantum_program(
            program,
            stack_qec_error,
            global_instructions={'Error': error},
            name=f"{gate[1].upper()} error on qubit {qubit}, FT Prep -, Error, QEC, measure X")

        program_qec_error.run(shots=100)

        from collections import Counter
        print(Counter(program_qec_error.collect_shot_data("logical_measurement", -1)))

# Again, we have some unexpected errors where we are not correcting errors
```
